## 1. What the app is (one paragraph)

A client-side, Brilliant-style calculus course (React 19 + TS + Vite 6 +
Tailwind 4). Lessons are version-controlled JSON, graded **synchronously in the
browser** by `feedbackEngine` (math.js). Firebase Auth + Firestore provide
identity/persistence, with a zero-config `localStorage` demo fallback. An
**optional** AI concept tutor is proxied through a Cloudflare Worker
(`tutor-proxy/`). The whole thing is ~131 first-party TS/TSX files in `src/`
(~17k lines) plus two backends.

### Tight coupling / inversion smells

| Smell | Where | Note |
|-------|-------|------|
| Non-AI feature → AI module | `solutionService.ts` imports `describeCorrectAnswer` from `aiTutor.ts` | Answer-to-text formatting lives in the tutor module; "Solve it" shouldn't depend on the tutor. Extract a neutral `answerFormat` module. |
| Domain types → UI registry | `types/content.ts` → `common/icons.ts` (`IconName`) | Move `IconName`/registry typing to a neutral location. |
| Duplicated predicate | `masteryService.isLessonDone` ≈ `progressService.isLessonComplete` ≈ `ProgressContext.isLessonDone` | Same `status === "complete" || "mastered"` in 3 places. |
| Prompt duplication across backends | `functions/src/tutor.ts` vs `tutor-proxy/src/prompt.ts` | Character-identical system prompts + near-identical validation/rate-limit. |

### Duplication inside the UI (from the component pass)

- `shuffledIndices(n)` copied verbatim in `MultipleChoiceInput` + `MultiChoiceInput`.
- Pointer drag/tap infra (~150 parallel lines) in `MatchInput` + `DragDropInput`.
- `prefersReducedMotion()` reimplemented in 3 animation files (and inline in `LessonPlayer`).
- Power-term → LaTeX formatting reimplemented in `AnswerInput`, `PowerRuleAnimation`, `polynomialBeats`.
- Plot constants (`W=340`, `PAD_*`) + Riemann SVG math duplicated in `RiemannInput` + `RiemannRefineAnimation` (neither uses the shared `plotGeometry.ts`).

- 5 practice/review pages (`PracticePage`, `ReviewPage`, `CustomPracticePage`, `LevelReviewPage`, `TestOutPage`) duplicate the same session-page state machine (~400 lines).

## 4. Efficiency observations

| # | Finding | Evidence | Proposed fix (behavior-preserving) |
|---|---------|----------|------------------------------------|
| E1 | **No code splitting** — one 2.25 MB JS chunk (gzip 609 kB) | `vite build` warning; `App.tsx` eagerly imports all 12 pages | `React.lazy` per route + `manualChunks` for `mathjs`/`katex`/`firebase`. Biggest user-facing win. |

# Part 2 — Executed changes (decisions: delete `functions/`, full structural scope, leave docs)

Every batch below ended green on the gate (`lint` + `build` + `test`, plus
`validate:lessons` where relevant). The starting **302 tests stayed at 302
passing** the whole way (no behavior change), `madge` still reports **no circular
dependencies** (144 files), and a browser smoke test confirmed the
landing/roadmap/lesson/grading/practice/solve-walkthrough flows work with **no
console errors**.

## Batch 1 — Dead code (safe set)
- Removed `TUTOR_MODEL` (stale `"gemini-2.5-flash"` constant; the live Worker uses gpt-4o-mini) — `aiTutor.ts`.
- Removed `isSoundEnabled()` (no callers) — `sound.ts`.
- Removed `ProgressContext.refresh` from the public context value + type (no consumer ever called it; the internal loader stays).
- Dropped unnecessary `export` on internal-only fns: `blendReviewRatio`, `masteryDecayMultiplier` (`masteryService`), `niceStep` (`plotGeometry`), `stepAsPart` (`content`/now `stepHelpers`), `termBody` (`polynomialBeats`).

## Batch 2 — Deleted the dead `functions/` backend
- Removed the entire `functions/` directory (Firebase Cloud Functions) — superseded by the Cloudflare Worker (`tutor-proxy/`), which is what the frontend calls. Zero importers in `src/`.
- Removed the `functions` block from `firebase.json`. (Left `firestore.rules` as-is: its `aiUsage`/`config` deny rules are harmless defensive config.)

## Batch 3 — DRY `feedbackEngine.checkAnswer` (efficiency/readability)
- **Before:** the identical wrong-answer object literal was repeated ~18× (and the success literal ~18×).
- **After:** three helpers — `pass(step)`, `fail(step, message?)`, `verdict(step, ok)` — drive every branch. File went ~500 → ~360 lines with **identical** grading output (verified by the 611-line `feedbackEngine.test.ts`).

## Batch 4 — Memoization (efficiency, behavior-preserving)
- Cached `getPublishedLessons()`, `getPracticeBank(lessonId)`, and `getLevels()` in `contentLoader.ts` — they derive purely from frozen import-time JSON yet were recomputed on nearly every render/session build. Same outputs; fewer allocations/sorts.

## Batch 5 — Decouple + dedup
- **Extracted `lib/answerFormat.ts`** (`describeAnswer`/`describeCorrectAnswer` + helpers) out of `aiTutor.ts`, so the non-AI "Solve it" feature (`solutionService`) no longer depends on the tutor module.
- **Unified the lesson-done predicate** into `lib/lessonStatus.ts` (`isLessonComplete`), shared by `progressService` and `masteryService` (previously duplicated; couldn't share before without a cycle).
- **Deduped UI helpers:** `lib/reducedMotion.ts` (was copied in 5 files), `lib/shuffle.ts` (`shuffledIndices`, copied in 2), and a single exported `QUOTA_MESSAGE` (copied in 2).

## Batch 6 — Fixed the type→UI inversion
- Moved the `IconName` vocabulary to `types/icons.ts`; the UI registry (`components/common/icons.ts`) now conforms to it via `satisfies Record<IconName, LucideIcon>` (so they can't drift) and re-exports it. `types/content.ts` no longer imports "up" into components.

## Batch 7 — Split the `types/content.ts` god file (~1,640 → ~1,300, types-only)
- Moved runtime code into focused modules: `lib/stepHelpers.ts` (step functions), `lib/milestones.ts` (milestone defs/sections/order + `milestoneProgress` + their types), `lib/constants.ts` (all tuning constants + `DEFAULT_ASSISTANCE_LEVEL`).
- Repointed ~20 importers directly to the new modules (no re-export barrel, to keep the graph acyclic). `content.ts` is now pure types.

## Batch 8 — Shared session-page machinery (DRY, ~5 pages)
- Extracted `hooks/useQuizSession.ts` (the result/attempt → re-sampled steps → synthetic lesson → exit-guard → results/retry state machine) and `components/lesson/SessionRunner.tsx` (the player + exit dialog shell).
- `PracticePage`, `ReviewPage`, `LevelReviewPage`, `CustomPracticePage`, `TestOutPage` now share them while keeping their own guards/intros/pickers/results. ~400 duplicated lines collapsed.

## Batch 9 — Decomposed `LessonPlayer` (~1,690 → ~1,400)
- Extracted pure helpers → `lessonPlayerHelpers.ts` (`seedAnswer`, `predictStartX`, `graphInitial`, `lerp`, `isTweenable`, `lerpAnswer`, …), the `LockedPart` subcomponent → `LockedPart.tsx`, and practice scoring → `usePracticeScore.ts` (narrow, deterministic interface).
- **Deliberately left** the solve-walkthrough animation state machine in place — see *Deferred* below.

## Batch 10 — `GraphWidget`
- Un-exported the unused `GraphReveal` type.
- **Deferred** the SVG render-split — see below.

## Batch 11 — Route-based code splitting (efficiency E1, the biggest user-facing win)
- `App.tsx` now `React.lazy`-loads every page behind a `Suspense` boundary; `vite.config.ts` `manualChunks` splits `mathjs`/`katex`/`firebase` into their own cacheable chunks.
- **Before:** one **2,251 kB** JS chunk (gzip **609 kB**) downloaded up front.
- **After:** initial load no longer pulls `mathjs` (656 kB), `katex` (274 kB), or `LessonPlayer` (114 kB) — those load only when a lesson/practice page is opened. Per-page chunks are 1–18 kB.

## Deferred (intentionally not done — would risk behavior without tests)
The components have **no unit tests** (coverage is scoped to `src/lib/**`), and
"all changes must be behavior-preserving" is the prime directive, so two deeper
splits were left as recommended follow-ups (best done after adding component/
visual tests):
1. **`LessonPlayer`'s solve-walkthrough hook** (`useSolveWalkthrough`): the
   tween/beat animation state machine. Extracting it is a wide-interface move
   that doesn't reduce coupling and risks subtle timing/stale-closure bugs.
2. **`GraphWidget`'s SVG render decomposition**: ~700 lines of intertwined
   coordinate math + drag handlers + reveal animations; splitting risks visual
   regressions that no test would catch.

## Remaining `knip` noise (intentionally kept)
- `tutor-proxy/src/*` reported as "unused files" — **false positive**: it's the live Cloudflare Worker, deployed separately (wrangler), not imported by `src`.
- `src/react-katex.d.ts` — required ambient types (do-not-remove).
- 16 "unused exported types" in `content.ts` — union members (e.g. the `AnswerSpec`/`SolveAnimation` arms); kept as the documented public domain vocabulary.
